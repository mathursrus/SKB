## Key Decisions
- **Static SVG over dynamic endpoint**: Simpler, zero runtime dependencies, cacheable. Dynamic endpoint deferred per spec.
- **`qrcode` npm as dev dependency**: Used for reproducible generation via `scripts/generate-qr.ts`. Not a runtime dependency.
- **Error correction level H**: Per spec recommendation for door-mounted durability.

## Deferrals
- Custom domain migration: QR SVG must be regenerated if production URL changes.
- Multiple print formats (table tent, takeout sticker): Not in scope for issue #2.
- QR analytics: No tracking parameters in v1 per spec.
